Keyboard Navigation Checklist

  • Navigation with Tab follows logical order
  • Focus indicators clearly visible on all interactive elements
  • Enter / Space activate buttons and controls
  • Esc closes modals and dropdowns
  • Skip to content link available for screen readers
  • No keyboard traps (can always tab away from elements)
  • All tool functionalities accessible without mouse

ARIA & Semantic HTML Checklist

  • All form inputs have associated <label> elements
  • Buttons use <button> not <div> with click handlers
  • Images have descriptive alt attributes
  • Icon-only buttons have aria-label or sr-only text
  • Error messages linked with aria-describedby
  • Loading states announced with aria-live regions
  • Page language declared (<html lang="...">)

Color & Contrast Requirements

WCAG 2.1 AA requires:

  • Normal text: 4.5:1 contrast ratio
  • Large text (18pt+): 3:1 contrast ratio
  • Interactive elements: 3:1 contrast ratio

Quick VoiceOver Test (macOS)

  1. Press Cmd+F5 to enable VoiceOver
  2. Navigate with Ctrl+Option+→
  3. Verify all content is announced clearly
  4. Check form labels and button purposes are clear
  5. Press Cmd+F5 again to disable

Integration with Playwright

For automated accessibility testing in E2E tests:

npm install --save-dev @axe-core/playwright

// In playwright.config.ts
import { test } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';

test('should not have accessibility violations', async ({ page }) => {
  await page.goto('https://webdev-tools.local/');
  
  const accessibilityScanResults = await new AxeBuilder({ page })
    .withTags(['wcag2a', 'wcag2aa', 'wcag21aa'])
    .analyze();
  
  expect(accessibilityScanResults.violations).toEqual([]);
});
