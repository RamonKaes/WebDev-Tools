External Resources Test

This page tests if all external resources are properly whitelisted in the CSP.

Expected Allowed Sources

  • script-src: 'self', 'nonce-*', https://cdn.jsdelivr.net, https://www.googletagmanager.com
  • style-src: 'self', https://cdn.jsdelivr.net
  • font-src: 'self', https://cdn.jsdelivr.net
  • img-src: 'self', data:, blob:, https:
  • connect-src: 'self' (default)

How to Monitor in Production

  1. Open Browser DevTools (F12)
  2. Go to Console tab
  3. Look for CSP violation errors (red text with "Content Security Policy")
  4. Check Network tab → filter by "blocked" to see blocked requests
  5. Use CSP report-uri for automated monitoring (optional)

Example CSP Violation:

Refused to load the script 'https://example.com/script.js' because it violates 
the following Content Security Policy directive: "script-src 'self' 'nonce-...'"

Production Checklist

  • ✓ QR Code Generator uses cdn.jsdelivr.net (allowed)
  • ✓ Bootstrap CSS from cdn.jsdelivr.net (allowed)
  • ✓ Bootstrap Icons from cdn.jsdelivr.net (allowed)
  • ✓ Google Tag Manager allowed (if used)
  • ✓ Inline scripts use nonce attribute
  • ✓ No unsafe-inline or unsafe-eval
  • ✓ Frame-ancestors set to 'none' (clickjacking protection)
  • ✓ HTTPS upgrade enabled in production
